## 一、本阶段目标（功能拓展）

在「多数据源 + 菜单化」骨架上，做**可演示的企业向能力与体验深度**，优先面试/产品叙事价值高、与现有架构契合的扩展。

---

## 二、功能拓展路线图

```
F1 数据源体验深化 → F2 问数能力深化 → F3 协作与治理 → F4 可观测与评测产品化
     1–2 周                1–2 周              2 周                 持续
```

### F1 — 数据源体验深化（优先）

| ID   | 任务           | 说明                                            | 验收              |
| ---- | -------------- | ----------------------------------------------- | ----------------- |
| F1-1 | Schema 浏览器  | 数据源详情页：表列表 / 列类型 / 行数预估 / 样例 | 点表可预览        |
| F1-2 | 连接向导       | 新建 MySQL/PG 分步：填连接 → 测试 → 保存        | 引导式不报裸错    |
| F1-3 | 方言提示模板   | 按 dialect 切换 NL2SQL system prompt 片段       | MySQL/PG 示例可跑 |
| F1-4 | 上传绑定数据源 | CSV 上传可选目标源（仅 SQLite 内置先支持）      | 文档说明清晰      |

### F2 — 问数能力深化

| ID   | 任务                   | 说明                                      | 验收                   |
| ---- | ---------------------- | ----------------------------------------- | ---------------------- |
| F2-1 | 数据源级知识库分区     | knowledge 按 `datasource_id` 或业务域隔离 | 切换源后 few-shot 不串 |
| F2-2 | 查询结果表格增强       | 可排序、分页、复制 SQL                    | 消息内表格交互         |
| F2-3 | 追问建议增强           | 基于当前 schema + 结果字段生成            | 推荐可点且可答         |
| F2-4 | Agent 过程时间线       | 工具耗时、SQL 次数、总 tokens（若可得）   | 流程追踪信息更全       |
| F2-5 | DATA_ANALYSIS 意图落地 | 分析类问题附加洞察段落                    | 与普通查询有差异       |

### F3 — 协作与治理

| ID   | 任务              | 说明                                    | 验收              |
| ---- | ----------------- | --------------------------------------- | ----------------- |
| F3-1 | 轻量用户/会话归属 | 可选登录或 `user_id` 头；对话按用户隔离 | 多浏览器不串历史  |
| F3-2 | 分享对话链接      | 生成只读分享 token                      | 外链可打开历史    |
| F3-3 | 审计增强          | 审计筛选、导出 CSV、按数据源统计        | 设置/数据源页可用 |
| F3-4 | 危险操作二次确认  | 删数据源、重建索引确认框统一            | 防误触            |

### F4 — 可观测与评测产品化

| ID   | 任务               | 说明                                  | 验收                   |
| ---- | ------------------ | ------------------------------------- | ---------------------- |
| F4-1 | 评测控制台页       | 跑 EvalRunner、展示准确率/执行匹配    | 前端可见数字           |
| F4-2 | 健康看板           | 模型/向量库/各数据源连通性            | 设置页或运维页         |
| F4-3 | 依赖弃用治理       | HuggingFaceEmbeddings / Chroma 包迁移 | 启动无弃用警告（尽量） |
| F4-4 | E2E 检查清单自动化 | 关键 API 冒烟脚本                     | `scripts/e2e_smoke.py` |

## 三、非目标（本版不做）

- 完整多租户 SaaS 计费
- 模型微调 / 私有化训练
- 语音输入、定时报表推送（可列入更远期）
- 替换前端框架

---

## 四、技术约束（延续）

- Python：conda 环境 **`wenshu`**
- 默认数据源：内置 SQLite 电商库
- 查询：只读 SELECT + 审计
- 密钥：`llm_settings.json` / 数据源密码加密，勿提交 git

---

## 五、建议执行顺序

1. **F1-1 + F1-2 + F1-3**（数据源可用感）
2. **F2-2 + F2-4 + F2-5**（问数体验）
3. **F4-1 + F4-4**（可量化叙事）
4. **F3-1 / F3-3**（治理）
5. 其余按需
